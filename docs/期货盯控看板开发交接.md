# 期货盯控看板开发交接

## 1. 项目定位

本项目当前定位为：

> 基于天勤行情 API 的期货量价/技术面盯控看板。

它不是交易下单系统，也不是自动给买卖建议的策略系统。当前版本主要回答：

- 哪些活跃主力合约值得盘后重点关注？
- 为什么值得关注？
- 当前属于趋势、突破、波动扩张、反转预警，还是中性状态？
- 这个异动更像小行情、中级行情，还是大行情候选？
- 后续需要观察什么确认条件？
- 什么情况下当前判断失效？

当前已明确暂缓：

- 基本面数据
- 消息面数据
- 宏观数据
- 社区舆情
- 自动交易建议
- 自动定时任务

## 2. 数据边界

当前只使用：

- 天勤实时行情
- 天勤历史行情
- AkShare 补充合约规格

曾测试过天勤 EDB 非价量数据，但当前账号可以生成 EDB token，调用指标查询接口时返回：

```text
error_code: 401
您的账户不支持下载非价量数据功能，需要购买后才能使用。
```

因此 EDB 暂不接入。

消息面也测试过金十、SHMET、百度经济日历、东方财富、Mysteel、期货日报等公开源。结论是：公开源要么噪声较大，要么稳定性/版权/登录问题较多，暂不接入项目。

## 3. 使用方式

项目目录：


启动：

```bash
cd /Users/damon/Desktop/期货盯控看板

```

浏览器打开：

```text
http://127.0.0.1:8011
```

```bash

```

账号配置在本地 `.env`：

```bash
TQ_ACCOUNT=天勤账号
TQ_PASSWORD=天勤密码
```

`.env` 已加入 `.gitignore`，不要提交。

## 4. 刷新模式

当前采用手动模式：

- 启动服务本身不会自动请求天勤。
- 打开页面时，优先读取当天缓存。
- 如果当天已有缓存，直接展示，不重复请求天勤。
- 如果没有当天缓存，页面会请求一次刷新。
- 用户点击“刷新数据”按钮时，强制重新请求天勤。

缓存文件：

```text
data/latest_snapshot.json
```

## 5. 盯控对象

当前盯控对象是活跃主力连续合约，按板块分组。

技术分析使用主力连续：

```text
KQ.m@SHFE.rb
KQ.m@DCE.i
KQ.m@CZCE.SA
...
```

同时展示当前实际主力合约和最新价。

低流动性品种会根据近 20 日成交量和持仓量自动剔除。当前默认门槛在 `analysis.py` 中：

```python
MIN_AVG_VOLUME_20 = 1000
MIN_AVG_OI_20 = 1000
```

## 6. 板块分组

当前在 `contracts.py` 中维护板块和合约池，主要包括：

- 黑色
- 有色
- 能化
- 农产品
- 软商品
- 贵金属
- 金融
- 航运

展示时既有全市场关注榜，也保留板块归属。

## 7. 周期口径

当前只使用三个周期：

- 周线：背景判断
- 日线：核心评分和状态判断
- 1 小时线：日内结构确认

之前讨论过 15 分钟，但用户盘后复盘为主，因此已删除 15 分钟周期。

暂不使用 1 分钟数据。

## 8. 交易日口径

期货夜盘归属下一个交易日：

- 夜盘算入下一个交易日。
- 周五夜盘算入下周一交易日。
- 遇节假日，归入节后首个交易日。

当前项目主要依赖天勤返回的 K 线交易日口径，后续如做更严格的交易日归档，需要单独处理交易所日历。

## 9. 核心判断逻辑

当前逻辑不是多维度简单加总，而是：

> 先判断主市场状态，再在状态内部打关注分。

状态互斥，避免一个品种同时被判断成趋势和震荡。

当前主要状态：

- 日线向上突破确认中
- 日线向下突破确认中
- 日线趋势上行
- 日线趋势下行
- 趋势上行后的反转预警
- 趋势下行后的反转预警
- 波动扩张但方向未确认
- 震荡收敛等待方向
- 中性无明确信号

关注分代表“盯控优先级”，不是买入/卖出分数。

## 10. 指标和依据

当前主要计算：

- 5/20/60 均线
- 14 日 ATR
- 日涨跌幅
- 日振幅
- 近 60 日成交量分位
- 近 60 日 ATR 分位
- 近 60 日振幅分位
- 近 120 日布林带宽度分位
- 近 20 日高低点突破
- 近 60 日高低点突破
- 持仓变化
- 1 小时结构路径
- 周线背景
- 同板块同方向品种数

输出必须可追溯到原始数据和公式，不做黑箱判断。

## 11. 行情级别判断

项目里不单独使用“消化力度”作为主概念，而是判断行情级别：

- 暂无行情级别
- 小行情或变盘前状态
- 小行情，需观察能否扩展
- 中级行情候选
- 大行情候选，仍需后续确认

判断依据包括：

- 是否突破 20/60 日区间
- 成交量是否处于高分位
- 持仓是否配合
- 1 小时结构是否连续推进
- 周线背景是否配合
- 同板块是否共振

## 12. 输出口径

看板不输出“买入/卖出”，也不输出空泛标签。

当前每个品种的详情页包括：

- 合约基本信息
- 开仓成本/合约规格
- 量价分析
- 后续可能成立的条件
- 判断失效的条件
- K 线图

量价分析把状态、行情级别、核心指标、诊断依据、后续可能情况合并为一个模块。

## 13. 合约成本和规格

详情顶部展示：

- 一手吨数/乘数
- 最小变动价位
- 一跳金额
- 预估保证金/手
- 一手合约价值
- 近 5 日日均波动/手

其中：

```text
一跳金额 = 最小变动价位 * 合约乘数
一手合约价值 = 最新价 * 合约乘数
近5日日均波动/手 = 最近5日平均(high - low) * 合约乘数
```

保证金优先使用天勤返回字段；如果天勤字段为空，则使用 AkShare `futures_fees_info()` 补充。

## 14. K 线图

K 线图放在详情页最下方，作为直观校验，不作为主信息。

当前支持切换：

- 日线
- 1 小时
- 周线

图上包含：

- 蜡烛图
- 成交量柱
- MA20
- MA60

## 17. 后续开发建议

短期建议继续打磨量价模块，不急着接消息面。

优先级建议：

1. 优化活跃品种筛选门槛。
2. 增加板块热力图和板块共振解释。
3. 做历史刷新结果留存，用于复盘。
4. 增加“昨日关注榜今天表现”。
5. 优化主力换月标记。
6. 对每类状态做复盘统计，调整阈值。
7. 后续如果有稳定 API，再接消息面或基本面。

## 18. 暂缓原因说明

消息面暂缓原因：

- 金十可抓，但泛资讯噪声大。
- SHMET 对有色/贵金属可用，但覆盖不全。
- 东方财富、Mysteel、新浪等黑色源能抓到内容，但稳定性、版权和结构化程度不够理想。
- 同花顺期货通社区不适合首版抓取，登录、JS、风控和噪声问题较多。

基本面暂缓原因：

- 天勤 EDB 当前账号无权限。
- AkShare 等公开数据可以补一部分，但需要单独设计指标体系。

当前最好保持项目边界：

> 先把量价/技术面的盯控体验做好，再决定是否接入消息面和基本面。
